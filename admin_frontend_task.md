# Задача: Реализация управления AI Ассистентами в Админ-панели

Необходимо реализовать раздел управления AI ассистентами в административной панели.

## 1. Общее описание
Раздел "AI Ассистенты" позволяет администратору создавать, редактировать и удалять настройки виртуальных помощников, доступных агентам. Каждый ассистент имеет имя, системный промпт (контекст) и модель ИИ.

## 2. API Методы (см. `docs/OPENAPI_SPEC.yaml`)
Все запросы должны быть авторизованы (как и остальная часть админки).

- `GET /api/admin/ai-assistants` - Список всех ассистентов.
- `POST /api/admin/ai-assistants` - Создание нового.
- `PUT /api/admin/ai-assistants/{id}` - Редактирование.
- `DELETE /api/admin/ai-assistants/{id}` - Удаление.

## 3. Экраны и Функционал

### 3.1. Список Ассистентов (Главный экран раздела)
- **Таблица** с колонками:
  - **ID**
  - **Название** (name)
  - **Slug** (уникальный код, например `ai-crm`)
  - **Модель** (например `google/gemini-2.0...`)
  - **Активен** (да/нет)
  - **Действия** (Редактировать, Удалить)
- **Кнопка "Создать ассистента"** вверху справа.

### 3.2. Модальное окно / Страница Создания и Редактирования
Форма должна содержать следующие поля:

1.  **Название (Name)**: Текстовое поле. *Пример: "Помощник по продуктам"*
2.  **Slug**: Текстовое поле (латиница, без пробелов). *Пример: `product-helper`*
3.  **Модель (Model)**:
    - Выпадающий список или текстовое поле с пресетами.
    - *Дефолтное значение*: `google/gemini-2.0-flash-exp:free`
    - Желательно дать возможность ввести свой ID модели вручную.
4.  **Системный Контекст (Context Template)**:
    - Большое текстовое поле (Textarea) или Markdown редактор.
    - **Важно**: Подсказка для администратора: "Используйте `{{agent_name}}` для подстановки имени агента в текст."
    - *Пример*: "Ты опытный наставник. Твоего собеседника зовут {{agent_name}}..."
5.  **Активен (Is Active)**: Чекбокс/Свитчер.

### 3.3. Логика
- При удалении — спрашивать подтверждение ("Вы уверены? Это удалит историю переписки агентов с этим ассистентом").
- Валидация: Slug должен быть уникальным.

## 4. Технические детали
- Использовать существующие компоненты UI (таблицы, модалки).
- Обработка ошибок: выводить сообщения от API (например, если slug занят).
